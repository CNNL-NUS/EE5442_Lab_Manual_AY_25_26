NeuroSim access on HPC

After logging in into the HPC following the steps from Lab 1 (Week 2). You will have to load the Pytorch container and also clone the updated NeuroSim from Seeder Research group.

  1. Calling the Pytorch container.
    /apps/containers/scripts/nntools/loadNNTools_Ub22.sh -shell --nv

  1. Loading the Pytorch environment.
    . /etc/profile
module load pytorch/py3/2.2.0

  1. Go into your “imem0XX” folder and then perform the below clone operation, as it creates a folder with the NeuroSim files collected from the web (Use cd command to enter into your directory).
  1. Clone the DNN NeuroSim repository from Seeder Research Group GitHub repository.
    git clone https://github.com/seeder-research/DNN_NeuroSim_V2.1

  1. Accessing the NeuroSim Manual PDF.
    DNN_NeuroSim_V2.1/Documents/DNN NeuroSim V2.1 User Manual.pdf

  1. You will need to familiarize yourself with the methodology used by NeuroSim by
    reading the user manual.
  1. The parameter file containing the definitions of the compute-in-memory arrays (which you need to modify) is the file below.
    DNN_NeuroSim_V2.1/Training_pytorch/NeuroSIM/Param.cpp

  1. The Python files that contain the other hardware constraints are listed below. These are files to be edited to change the NN, hyperparameter tuning and other knobs.
    DNN_NeuroSim_V2.1/Training_pytorch/inference.py
DNN_NeuroSim_V2.1/Training_pytorch/train.py

  1. Before running a simulation, you will need to go into the directory “DNN_NeuroSim_V2.1/Training_pytorch/NeuroSIM” and run the command.
    To ensure that the edited C++ files are complied before running the python script for training and inference. For more clarity please read through the DNN NeuroSim Maunal PDF.
    make clean && make

  1. To run the python script you can run python3 on command line, or submit it as a job to the server. The steps are discussed in Lab 1 (Week 2) on submitting jobs to the HPC.

    Submitting as a job is preferred to the command line invoke. The submit and wrapper shell script files can be used.
