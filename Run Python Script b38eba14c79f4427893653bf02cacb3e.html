Run Python Script

A walkthrough to run a simple python script to get familiarized with the container and editor. Here an interactive terminal based commands are utilized.

  1. Login to the HPC server from MobaXterm with the given credentials. If outside NUS please utilize the nVPN softwares to connect. HPC Access page can be used for reference.

  1. After establishing your SSH session with either Feynman or Schrodinger, run the following command to get a terminal in one of the other compute servers. It will allot you a compute node based on the resources available.
    qsub -I -l ncpus=8 -l mem=8gb -l walltime=11:00:00 -l cudamem=1gb

  1. Select the container to run, one for the workstation and the later is for the server cluster and to run on interactive shell (-shell) is used. For the Nvidia drivers (--nv) is utilized.
    /apps_SEEDER01/containers/scripts/nntools/loadNNTools_Ub22.sh -shell --nv

OR

/apps/containers/scripts/nntools/loadNNTools_Ub22.sh -shell --nv

  1. To list all the install modules inside the container and then load the required modules, in this case loading pytorch version 2.2.0 under Python3.
    .  /etc/profile
module av
module load pytorch/py3/2.2.0

  1. Create a directory inside the imem0XX folder, named “trial”. And create a new text file and rename it as “try.py”. With the support from the FTP interactive explorer creating the directory and text file would be easier.

    If using the command line the below instructions can be used. (Alternatively Vim, Nano or Touch commands can also be used to edit the file and save them as “py” extension).

    Once the try.py file is opened in Vim, press “i” for insert mode to edit the scrip and then “Esc” to enter the commands mode inside Vim for “:wq” to save and exit.

    mkdir ~/trial && cd ~/trail
vim try.py

  1. The python script is kept simple to test the library invoking, and compiler working.
    import numpy as np
import torch
import torch.quantization
import torch.nn as nn

print("Test is successful")

device = torch.device('cuda' if torch.cuda.is_available() else 'cpu')
print("Device available CPU/GPU: ", device)

  1. Running the python script from the interactive terminal and validating the results.
    python3 try.py

Submit the Python script as a job to the HPC

  1. Two shell files as introduced in the HPC jobs submission are required.
  1. The two shell files are described below, tailor made for running python scripts. The files can be downloaded and added into the “imem0XX” folder for submitting multiple jobs to the HPC.
  1. This feature overcomes the shell closing timer, which will break the running of the program if the compute time is too long. The results and log files will be created automatically once the job is completed.
  1. When multiple jobs are submitted to the same node, they will run in a queue based on the resources available.
#!/bin/bash
#PBS -j oe
#PBS -l vnode=Landauer47
#PBS -l walltime=24:00:00
#PBS -l cudamem=500mb
#PBS -l ncpus=4
#PBS -l mem=8gb


export working_dir='/users/----/----/----/'
export path_to_wrapper_script='/users/----/----/----/'wrapper.sh'

# For this script to execute somthing meaningful, replace
# <working_dir> and <path_to_python_script> with the
# absolute paths to 1) the directory in which the last
# command is to be run from, and 2) the wrapper script to
# run. Then, uncomment the next five lines.
cd ${working_dir} && \
/apps/containers/scripts/nntools/loadNNTools_Ub22.sh \
  -run \
  --nv \
  ${path_to_wrapper_script}

Personal Computer

Students can also use the personal computer for practice (final code submission should be done on the HPC as per the grading requirements).

Students can install Python and other dependencies based on the requirement. Numerous sources are available online for guidance. IDEs such as PyCharm or Anaconda environment can be installed for easier access. Jupyter notebooks can be used for interpreter style coding with blocks, while making it into a reference notebook.
