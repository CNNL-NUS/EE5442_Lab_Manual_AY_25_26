Run jobs inside Singularity Containers

The server is setup to run the Singularity Containers, giving flexibility to run software behind the environments configured specially for each. The example scripts are provided for reference in accessing such software on the HPC system.

In the Labs the focus would be on using ‚Äúnntools‚Äù, ‚Äúumag‚Äù, ‚Äúsentaurus‚Äù, ‚Äúneurosim‚Äù.

  1. A set of scripts have been developed on the cluster so that you may access these environments. They are found in this directory, where each directory has very specific environment setups
    /apps/containers/scripts/
    DirectoryDescription
    nntoolsEnvironments for running neural network frameworks in Python. Installed frameworks include PyTorch and Tensorflow.
    (A text file in the directory lists more details about the frameworks available).
    umagEnvironments for running micromagnetic software tools.
    These are primarily OOMMF and MuMax3.
    sentaurusEnvironment for running commercial grade EDA software tools.
    These are software from Cadence Design System, Synopsys, Mentor Graphics, Siemens and Xilinx.
    neurosimEnvironment for running neurosim.
    visualizationEnvironment for running visualization software tools. In the simulation cluster, these are ParaView and muview.
    openedaEnvironment for running open source EDA software tools.
    These are part of the OpenEDA community, including tools such as magic, netgen, ngspice, Xyce, yosys, etc.

  1. To get a terminal inside the container environment, please run the corresponding script with the "-shell" switch. For example, running the nntools container using the following command. The table below lists the switches that can be used with the scripts
    /apps/containers/scripts/nntools/loadNNTools_Ub22.sh¬† -shell
    SwitchDescription
    -shellReturns a terminal in the container environment
    -nvRuns a script or command inside the container environment
    -execExecutes a command inside the container environment

  1. Additional switches are needed to set up the environment to use an available GPU accelerator device Nvidia or AMD Instinct. At this time, Nvidia and AMD Instinct GPUs cannot be used concurrently. The table below lists the additional switches that can be used.
    Additional SwitchDescription
    --nvSets up environment to use an Nvidia GPU
    --amd Sets up environment to use an AMD Instinct GPU

Submit multiple jobs with shell script

  1. The above are commands to have interactive setup for the container and software access. The further commands or scripts are non interactive commands of OpenPBS, which can be utilized by running them on a shell script file. The code snips shown below are for your understanding, please use the shell files given at the end.

    For the script that is submitted to run a program inside a Singularity container, a wrapper script is needed. An example of a wrapper script is the following:
    #!/bin/bash
.  /etc/profile
module load local pytorch/py3/2.2.0 
cd /working_dir && \
python3  /python_script_filename.py

    (working_dir is the current project or script directory, for example it will be in the allotted ‚Äúimem0XX‚Äù for each student).

    This script can be saved as wrapper.sh shell file which will be called by the submit.sh, the submit script is given below. Here please updated the name of the server you want your python script to run on.

    
#!/bin/bash
#PBS -j oe
#PBS -l vnode=Server_Name
#PBS -l walltime=24:00:00

(Here nntools container is utilized to access the softwares related to the Neural Networks).
