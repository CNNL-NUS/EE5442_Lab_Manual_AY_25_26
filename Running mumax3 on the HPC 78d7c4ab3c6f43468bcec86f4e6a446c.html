Running mumax3 on the HPC

SSH Connection to the HPC

  1. Setup the SSH connection to the cluster.
  1. Login with your username and password.
  1. Access the files on the SSH Browser (FTP)
  1. The .ovf files, tables and log can be downloaded and analyzed on local machine. Use the imem0XX folder assigned to you from before.
  1. Create .mx3 files to run scripts on mumax3.

Load mumax3 interactive shell

  1. srun : batch client to submit scripts to server Using slurm to get allotted with a server of following settings.
    srun --pty --nodelist=boltzmann01 --cpus-per-task=4 --mem=8G --gres=cudamem:300 --time=4:00:00 bash

    No need for this step if you are using workstation, directly skip to step 2.

  1. Load the mumax3 module.
    module load mumax/3.11.1

    If you are using the workstations, kindly use this instead of above:

    module load mumax3/3.11.1

    Rest of the steps are same for workstation and cluster.

  1. Go to the directory with .mx3 file to run / add the relative path, and run with mumax3.
    mumax3 --cache $TEMPDIR test.mx3

To have web interface in Firefox browser, gives a local port. Open Firefox and load the address (127.0.0.1:port). For VPN (server_ip:port).

Web Browser interface is not being used in this lab.

mumax3 –i test.mx3

Submitting mumax3 jobs

  1. In longer simulations the host will logout the user after sometime. If running in interactive mode data is lost. Submitting the jobs via slurm scripts, gives a Job-ID and ensure the job runs in the background.
  1. The shell script run_single.slurm is provided. Place them in the running directory. The scripts can be downloaded here.
  1. Only one job to be submitted at a time, to allow balanced access to all groups.
  1. Change the script_4.mx3 file name according to your script file name.

  1. Edit the Project directory to your project path.
  1. Edit the file name in run_single. 
 sbatch run_single.slurm

  1. A Job-ID will be assigned. Check status.
squeue -u "user_name"

  1. Delete the job.
 scancel "job-id"
